Introduction Haut de page

Qu'est-ce que 'Sommaire Paramétrable' ?
Sommaire Paramétrable est un outil qui vous permet de contrôler totalement le contenu et le look de votre sommaire (le bloc de menu de votre site PHP-Nuke):

  • Menu séparé en catégories, avec une icone
  • Affichage des modules installés, avec une icone
  • Ajout de liens externes (n'importe quelle url), avec icone
  • Les liens externes pointant vers une page spécifique d'un module sont traités comme les modules (accès restreint avec icone 'interdit')
  • Menu dynamique (contenu des catégories affiché/caché avec un clic)
  • Présentation des nouveautés du site :
    1. Ajout manuel d'une icone 'NEW' devant chaque catégorie et chaque module/lien
    2. Ajout automatique d'une icone 'NEW' devant les modules de base, avec la détection des nouveautés du site, et choix de la durée des "nouveautés".
  • Possibilité de mettre un lien ou une catégorie en gras
  • Contenu des catégories peut être affiché en listbox
  • Choix du style de chaque lien/module (classe css) et de chaque catégorie
  • Affichage possible des catégories avec uniquement une image
  • Affichage possible des liens/modules avec uniquement une image
  • Barre horizontale pour séparer 2 catégories ou 2 liens
  • Titre de catégorie centré
  • Couleur de fond différente pour chaque catégorie
  • Affichage d'une icône dans le sommaire si le membre a des Messages Privés non lus.
  • Masquer ou non aux visiteurs les modules réservés aux membres, avec gestion des groupes pour PHP-Nuke v.7+
  • Mettre un fichier FLASH .swf à la place du nom de catégorie (FLASH implémenté par MAC06 - magetmac06@hotmail.com)
  • Sommaire devrait fonctionner avec Google Tap NExtGEn (http://gt.audioslaved.com) avec GT-NExtGEn Main Menu Tweak.
  • et plus encore...

Installation Haut de page

Comment installer Sommaire Paramétrable ?
C'est très simple, suivez les indications suivantes :

  1. Téléchargez la dernière version de Sommaire Paramétrable (v2.1.1).
  2. Dézippez l'archive, en respectant l'arborescence.
  3. Uploadez le CONTENU du répertoire /html/*.* à la racine de votre site web PHP-Nuke.
    (install-sommairev21.php est à la racine du serveur, block-Sommaire.php est dans le sous-répertoire /blocks/ etc...)
  4. Lancer install-sommairev21.php avec votre navigateur et suivez les instructions.
  5. Le bloc "Sommaire" a été ajouté automatiquement dans votre base de données, vous pouvez maintenant désactiver le bloc "Modules" original.
    (Dans l'administration des blocs (admin.php?op=BlocksAdmin) vous pouvez définir le Poids des blocs avec les flèches. Vous pouvez également changer le titre du bloc "Sommaire" en cliquant sur 'Editer' dans l'admin des blocs.)
  6. Ensuite administrez votre sommaire en cliquant sur l'icone 'Sommaire' dans la console d'administration de PHP-Nuke (admin.php?op=sommaire)

Je ne veux pas utiliser le script d'installation automatique, comment installer manuellement ?
Si vous préférez effectuer les modifications de votre base de données et des fichiers de langue manuellement, suivez les instructions indiquées dans le répertoire /manual-install/ de l'archive du Sommaire.

L'installation automatique crée des fichiers de backup : dois-je les supprimer ?
Lors de l'installation, les fichiers de langue sont sauvegardés avant de tenter toute modification.
Les fichiers créés sont du type : lang-XXX.php.sommairebackup20040401
Si l'installation s'est déroulée sans message d'erreur, ces fichiers ne sont plus utiles et peuvent être supprimés sans problème. (ce choix vous est proposé à la fin de l'installation : cliquer sur le bouton correspondant...)
Si l'installation n'a pas pu se terminer correctement, certains fichiers de langues n'ont pas pu être modifiés, ou les modifications ne sont pas complètes.
Dans ce cas, la meilleure solution est de remplacer les fichiers lang-XXX.php par leur copies de sauvegarde (lang-XXX.php.sommairebackup20040401) et de faire les modifications des fichiers de langues manuellement (cf. le répertoire /manual-install/ de l'archive zip du sommaire).

Pour désinstaller complètement le sommaire, vous pouvez enlever les termes ajoutés dans vos fichiers de langue pendant l'installation.
Pour cela, vous pouvez tout simplement remplacer vos fichiers lang-XXX.php par les backups, ou bien (si vous avez supprimé ces fichiers) enlever manuellement ces termes.
(C'est simple : éditez les fichiers de langue, et allez à la fin du fichier : les termes ajoutés pour le sommaire sont clairement indiqués dans le code)

Note : Ces termes ne gênent en rien le fonctionnement de votre site web, même si vous n'utilisez plus Sommaire Paramétrable.

A propos des requêtes SQL...
Le code de la version 2.1 a été modifié pour diminuer le nombre de requêtes dans la base de données. Une réécriture complète du code du bloc a été nécessaire pour optimiser au maximum le nombre de requêtes et ajouter les nouvelles fonctionnalités.

Comparaison
La comparaison a été effectuée pour chaque situation : si la page est visualisée par un Visiteur, un Membre, un Administrateur non loggé en tant que membre, et un Administrateur étant également loggé en tant que membre.

  1. Bloc Modules original
    • Visiteur : 3 requêtes
    • Membre : 4 requêtes
    • Administrateur : 81 requêtes
    • Administrateur + Membre : 82 requêtes

  2. Sommaire Paramétrable v2.1
    • Visiteur : 4 requêtes
    • Membre : 6 requêtes
    • Administrateur : 7 requêtes
    • Administrateur + Membre : 9 requêtes
    Ajouter 1 requête pour PHP-Nuke v7+ : Sommaire Paramétrable effectue la gestion des Groupes d'utilisateurs.
    Ajouter 0 à 6 requêtes si l'option "Détection automatique des nouveautés" est activée.

  3. Sommaire Paramétrable v2.0
    • Visiteur : 7 requêtes
    • Membre : 30 requêtes
    • Administrateur : environ 300 à 400 requêtes
    • Administrateur + Membre : environ 300 à 400 requêtes

La différence avec le bloc Modules original est donc de seulement 1 à 3 requêtes supplémentaires ! (70 requêtes en moins quand on est un admin ;-) )
Cela est nécessaire pour activer toutes les fonctionnalité de Sommaire Paramétrable.
Si vous ne souhaitez pas bénéficier de la gestion des Groupes d'utilisateurs (à partir de PHP-Nuke 7.0), vous pouvez modifier la ligne 31 du block-Sommaire.php afin d'économiser une requête SQL (mettre $gestiongroupe = 0).
Si vous ne souhaitez pas bénéficier de la détection des Messages Privés, changez la valeur de $detectPM dans block-Sommaire.php à la ligne 32 (mettre 0), afin d'économiser une requête SQL.



Comment fonctionnent les "Liens externes" ?
Les "Liens externes" sont utiles pour créer un lien vers n'importe quelle adresse internet dans votre sommaire.
Pour utiliser cette fonctionnalité, sélectionnez " *Lien Externe* " dans la liste déroulante des modules, dans l'administration du sommaire.
Deux nouvelles zones de texte vont apparaître, où vous devez indiquer l'url du lien, ainsi que le texte qui doit apparaître dans le sommaire.

Ces liens externes peuvent être n'importe quelle adresse internet valide (lien vers un fichier, streaming, etc...).

NOUVEAU : Si le lien est une url pointant sur une page spécifique (module) de votre site web, l'accès à ce lien dans le sommaire sera traité comme le module parent.
Exemple : le module Downloads est réservé aux membres. un lien externe : "modules.php?name=Downloads&d_op=viewdownload&cid=1" qui pointe vers une catégorie spécifique du module Downloads sera invisible aux visiteurs anonymes, ou aura une icone 'interdit' devant, selon l'option choisie.
Note : Le lien DOIT commencer par "modules.php", et non par "http://www.votresite.com/modules.php" : c'est un lien relatif.



Comment ouvrir une page dans une nouvelle fenêtre internet, ou l'ouvrir dans la fenêtre courante ?
C'est simple : pour ouvrir une page dans une nouvelle fenêtre, le lien doit commencer par :
  • http://
  • https://
  • ftp://
  • lien relatif (lien direct vers une page du site web, sans spécifier le domaine, ex: "modules.php?name=Topics")
Pour ouvrir une page dans la même fenêtre, le lien doit commencer par :
  • HTTP://
  • HTTPS://
  • FTP://
Pour vous aider à visualiser rapidement si le lien va s'ouvrir dans une nouvelle fenêtre ou dans la même fenêtre, dans la console d'administration de SP l'icone devant le champ 'url' est modifiée pour représenter l'état nouvelle fenêtre/même fenêtre.


Comment ouvrir un lien en popup ?
Pour ouvrir un lien en popup, vous devez utiliser la fonction "Lien externe".
Dans le champ 'url' du lien externe, vous devez inscrire ceci :

javascript:window.open('http://www.url.com','popup_sommaire','directories=no,menubar=no,status=no,location=no,scrollbars=no,resizable=no')

Comment changer la couleur du texte (catégories, liens), la taille, la police, etc... ?
Vous pouvez définir l'apparence du texte de chaque catégorie et de chaque lien (module/lien externe).
Pour cela, vous devez utiliser les classes CSS. C'est très simple :
Les classes CSS permettent de définir l'apparence du texte : couleur, taille, police, etc...
Pour plus d'infos sur les classes CSS, vous pouvez notamment visiter ICI pour comprendre ce que sont les CSS, LA pour voir des exemples en pratique, LA pour la liste complète des propriétés des CSS, et enfin ICI (une série d'articles intéressants).
Voici un exemple pour les catégories :

.sommairecat {FONT-FAMILY: Verdana,Helvetica; FONT-SIZE: 12px; COLOR: #000000; FONT-WEIGHT: bold;}

Copiez cette ligne dans le fichier /themes/VOTRETHEME/style/style.css (éditable avec notepad ou un éditeur de code).
et revenez dans l'administration du sommaire : changez la "Classe utilisée pour les catégories" (en bas du tableau) pour indiquer "sommairecat". (la valeur par défaut est storytitle)

Voici un exemple pour les liens/modules du sommaire :

.sommairelink {FONT-FAMILY: Verdana,Helvetica; FONT-SIZE: 10px; COLOR: #000000;}

Même fonctionnement : coller la ligne dans le fichier style.css, et changez la classe CSS pour les liens/modules dans l'administration du sommaire. (la valeur par défaut est boxcontent)

Vous pouvez également définir un style différent pour chaque catégorie et chaque lien/module : cliquer sur " [+] " (plus d'options) dans l'administration du sommaire, et définir vos classes CSS dans la fenêtre qui apparait.



Comment utiliser d'autres images dans le sommaire?
C'est simple : ajoutez vos propres images dans le répertoire :
  • /images/sommaire/ pour les images des catégories
  • /images/sommaire/categories/ pour les images des liens/modules
Ces images vont automatiquement apparaître dans les listes déroulantes de l'administration du sommaire.

Pour changer l'icone 'NEW', il faut remplacer le fichier :
  • /images/sommaire/admin/new.gif
par l'image que vous voulez.
Vous pouvez essayer l'image /images/sommaire/admin/nouveau.gif (renommez-là en new.gif), qui est moins large que l'image NEW originale.



Des modules apparaissent en bas du bloc (non sélectionnés dans l'administration du sommaire), je ne veux pas les afficher !
Depuis la version 2.1, les modules non sélectionnés dans l'administration du sommaire ne sont visibles qu'aux administrateurs.
Ils sont regroupés en 3 groupes :
  • Modules actifs et visibles, mais non sélectionnés : Ce sont les modules devant normalement être affichés dans le sommaire, mais qui ont été "oubliés". Ils sont accesssibles aux membres ou visiteurs si ils tapent l'url.
  • Modules actifs et invisibles : Ce sont les modules qui ne doivent pas apparaître dans le sommaire (cela est défini dans l'administration des modules : Cliquer sur 'Editer' et cocher 'Visible dans le bloc des modules'). Ils sont accessibles par leur url mais le lien n'apparait pas dans le sommaire (MEME S'IL A ETE SELECTIONNE DANS L'ADMINISTRATION DU SOMMAIRE!).
  • Modules inactifs : Ce sont les modules inacessibles aux visiteurs et membres, même en tapant l'url. Seuls les Superadmins peuvent les utiliser.

Seuls les administrateurs voient ces 3 groupes de modules, c'est totalement transparent pour les visiteurs et les membres.



Comment fonctionne l'option "Détection automatique des nouveautés" ?
Cette option permet de détecter automatiquement les nouveautés pour les modules suivants :
  • Téléchargements (Downloads)
  • Liens Web (Web_Links)
  • Pages libres (Content)
  • Comptes Rendus (Reviews)
  • Journal (Journal)
  • Articles (News) uniquement si le module n'est pas en page d'accueil!
Les "nouveautés" sont détectées grâce au champ 'date' des tables de la base de données pour ces modules : si une entrée date de moins de XX jours (XX peut être défini dans l'administration du sommaire), l'icone 'NEW' apparait.

Si un 'Lien externe' pointe directement vers une page d'un de ces modules, la détection des nouveautés va également se faire !! (utile si vous créez des liens directs vers des catégories de downloads, etc...)
(pour cela, le lien ne doit pas commencer par 'http://' ou 'HTTP://', mais commencer directement par 'modules.php?name=')

J'ai un message d'erreur : 'Poids incorrect. Le Poids doit être compris entre 1 et 98', comment résoudre le problème ?
Ce message d'erreur peut avoir plusieurs causes.
  • Le POIDS est effectivement incorrect !
    La première cause, la plus évidente, c'est que vous avez indiqué un 'Poids' invalide pour une catégorie! Corrigez et réessayez.

  • Tables inaccessibles
    Ce message peut apparaitre parce que les tables du sommaire dans la base de données sont inaccessibles.
    Cela peut être dû à une mauvaise installation. Faites une installation manuelle (instructions dans l'archive zip du sommaire : dossier /manual-install/)
    Si vous avez opté pour l'installation manuelle, peut-être avez-vous oublié de modifier le préfixe des tables dans le fichier sql, avant son importation dans la base de données ?

  • Vous avez installé un addon avec un bug, interférant avec l'administration du sommaire
    Jusqu'à présent 3 addons posant problème ont été identifiés :
    1. Shoutblock (Résoudre le problème)
    2. un éditeur WYSIWYG (Résoudre le problème)
    3. un patch de phpSecure dans mainfile.php (parfois intégré d'office dans votre PHP-Nuke, selon la version et le site de téléchargement) (Résoudre le problème)

    Regardez les sections correspondantes dans la FAQ pour résoudre le problème.


Comment créer un menu Multilingue ?
Vous pouvez créer un menu multilingue avec Sommaire Paramétrable.
Les visiteurs qui choisissent une langue différente verront les noms de catégories et les modules/liens dans leur langue !

Pour cela, suivre attentivement ces instructions :
  1. Dans l'administration du sommaire il faut indiquer le nom des catégories comme suit :
    'LANG:_SOMCAT0' pour la catégorie 0,
    'LANG:_SOMCAT1' pour la catégorie 1, etc...

    (nota : vous pouvez écrire ce que vous voulez après 'LANG:_' il faut juste que vous vous y retrouviez et que le nom choisi ne soit pas déjà utilisé dans le fichier de langue !)

  2. Tous les modules devront être créés grâce aux 'Liens externes'.
    --> mettre comme adresse du lien l'adresse du module, c'est à dire 'modules.php?name=Nom_Du_Module'
    --> mettre comme texte du lien 'LANG:_NomDuModule' (ex. 'LANG:_SOMSURVEYS')

  3. Ensuite, éditer les fichiers de langues (racine du site /language/lang-xxx.php)
    --> rajouter une ligne pour chaque catégorie et module/lien à afficher :
    ex.
    define ("_SOMCAT0","Accueil");
    define ("_SOMCAT1","Contenu");
    define ("_SOMSURVEYS","Sondages");
    etc...
    --> Il faut recopier ces lignes dans les autres fichiers de langue, en traduisant les textes.
    ex. pour l'anglais ça donnera :
    define ("_SOMCAT0","Home");
    define ("_SOMCAT1","Content");
    define ("_SOMSURVEYS","Surveys");
    etc...

  4. Aller dans Administration->Modules, et éditer chaque module :
    décocher la case "visible dans le block des modules".
NB : NE PAS écrire 'LANG:' dans les fichiers de language ! uniquement ce qui suit 'LANG:'
NB2: Les fichiers de langue ne sont pas dans /admin/language/lang-xxx.php !! il faut bien éditer les fichiers de langue dans /language/lang-xxx.php

NOUVEAU : Vous pouvez maintenant créer des url différentes pour chaque langue ! (suivre les mêmes indications mais ajouter également le champ URL dans les fichiers de langue et dans le sommaire)

    Mon éditeur WYSIWYG pose problème avec Sommaire Paramétrable ! -->Solution
    Certains éditeurs wysiwyg peuvent entraîner des problèmes avec l'admin du sommaire, car ils modifient l'apparence (et le code) des formulaires hml, ce que le sommaire ne supporte pas.

    Dans un premier temps si le sommaire ne fonctionne pas, désinstallez complètement votre module wysiwyg. (enlever tous les fichiers ajoutés par celui-ci, et supprimer les modifications de certains fichiers demandées dans le fichier d'installation)
    ==> Si le sommaire fonctionne à nouveau, c'est bien le wysiwyg qui est la cause du problème ! donc réinstallez le wysiwyg et faites ces modifs :

    Les wysiwyg peuvent ajouter un fichier dans /includes/ et/ou vous faire modifier les fichiers header.php et footer.php
    Il faut encadrer le code de ces modifications par ça :
    if (!(eregi("admin.php", $_SERVER['PHP_SELF']) AND $_GET['op']=="sommaire")) {

    ICI LE CODE AJOUTE PAR WYSIWYG DANS LE HEADER, LE FOOTER, OU TOUT LE CODE PHP DU FICHIER AJOUTE DANS /includes/ .

    }


    Par exemple, un module wysiwyg vous demande de rajouter 2 lignes dans le header.php :

    include("includes/wysiwyg.php");
    WysiwygJscriptHeader();

    ce qui vous donne :
    function head() {
