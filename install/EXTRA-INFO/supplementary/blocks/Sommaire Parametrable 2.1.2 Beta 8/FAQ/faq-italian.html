Introduzione Torna all'inizio

Cosa è 'Sommaire Paramétrable' ?
Sommaire Paramétrable è un'utilità che permette di controllare totalmente il contenuto e la grafica del tuo Sommaire (il blocco menu del tuo sito PHP-Nuke):

  • Menu diviso in categorie con icone
  • Mostra i moduli installati, con un'icona
  • Mostra i links esterni (ogni url valido), con un'icona
  • I links esterni che puntano ad una pagina specifica vengono trattati come moduli (quando l'accesso ad un modulo è riservato, viene visulizzata un'icona di "divieto d'accesso')
  • Menu dinamico (Il contenuto delle categorie è visulizzato con un semplice click)
  • Visulalizza Nuovi contenuti nel sito :
    1. Aggiunta manuale di una 'NUOVA" icona in ogni categoria o modulo
    2. Aggiunta automatica di una 'NUOVA" icona in ogni modulo PHP-Nuke, con la scelta della durata della novità.
  • Opzione di visulizzare una categoria o un modulo/link in grassetto
  • Opzione di visulizzare una categoria o un modulo/link usando una listbox
  • Opzione di visulizzare una categoria o un modulo/link usando (CSS)
  • Le categorie possono essere visulizzate usando solo un'imaggine (no testo)
  • Moduli/Links possono essere visulizzate usando solo un'imaggine (no testo)
  • Può essere usata una linea orizzontale per separare due categorie o due moduli/links
  • Opzione di centrare il titolo delle categorie
  • Opzione di modificare il colore di sfondo della categoria
  • Visualizza l'icona di un "Nuovo messaggio Privato" agli utenti che non hanno letto I messaggi privati.
  • Opzione per nascondere agli anonimi i moduli che sono riservati a soli membri, con rilevazione dei gruppi di utenti PHP-Nuke v.7+
  • E' possibile usare FLASH .swf file (FLASH support by MAC06 - magetmac06@hotmail.com)
  • Sommaire funziona con Google Tap NExtGEn (http://gt.audioslaved.com) con GT-NExtGEn Menu Principale.
  • e molto altro ancora...

E' compatibile con tutti i browsers ?
Sommaire Paramétrable è compatibile con tutti i browsers.
Ma la funzione "menu dinamico" può essere visulizzata solo con i  browser più recenti.
Sommaire Paramétrable v2.1 è stato testato su Internet Explorer 6, Opera 7, Konqueror 3.1.4.
Gli utilizzatori di Mozilla non visulizzano correttamente la funzione "dinamica", quindi la funzionè è disabilitata automaticamente agli utilizzatori di Mozilla.
Per altri dettagli leggi la sezione Bugs.

Installazione Torna all'inizio

Come s'installa Sommaire Paramétrable ?
E' molto semplice, basta seguire queste indizazioni :

  1. Fai il downloads dell'ultima versione di Sommaire Paramétrable (v2.1).
  2. Decomprimi l'archivio della struttura in una cartella
  3. Invia sul server il contenuto rispettando la struttura del tuo sito PHP-Nuke .
    (install-sommairev21.php va inserito nella cartella html (la root) block-Sommaire.php è una sottocartella /blocks/ etc...)
  4. Lancia install-sommairev21.php con il tuo browser, e segui le istruzioni.
  5. Il blocco "Sommaire" è automaticamente aggiunto nel tuo database, ora puoi disattivare il tuo blocco originario "Modules".
    (nell'amministrazione blocchi (admin.php?op=BlocksAdmin) puoi scegliere il peso del blocco. Puoi anche modificare il titolo del blocco "Sommaire" cliccando se 'Modifica' nell'admin blocchi.)
  6. Ora puoi configurare il tuo menu! Clicca sull'icona 'Menu' in amministrazione di PHP-Nuke (admin.php?op=sommaire)

 
Non voglio usare lo script d'installazione, posso installarlo manualmente ?
Se preferisci effettuare la modifiche nel tuo database e nei file di lingua manualmente, segui le istruzioni del manuale /manual-install/readme.txt nel file Sommaire zip.

Lo script d'nstallazione crea un backup dei file di lingua, posso cancellarli ?
Durante l''installazione viene effettuato un backup dei file di lingua per prevenire eventuali modifiche.
I file creati avranno nomi come questi : lang-XXX.php.sommairebackup20040401
Se l'installazione va a buon fine  senza messaggi di errore, questi file ossono essere cancellati. (hai questa possibilità alla fine del processo d'installazione : cliccando sul pulsante corrispondente...)
Se l'installazione non va a buon fine alcuni file di lingua non possono essere modificati, o le modifiche sono incomplete.
In questo caso la migliore soluzione, è rimpiazzare I file di backup (lang-XXX.php.sommairebackup20040401) e effettuare le modifiche manualmente (le istruzioni sono in /manual-install/readme.txt , del file zip sommaire).

Se disinstalli completamente Sommaire Paramétrable,  puoi cancellare il testo aggiunto nel file language durante l'installazione.

Nota :Questo testo non interferisce con il tuo sito, anche se non userai più Sommaire Paramétrable.

Circa le interrogazioni SQL ...
Il codice del blocco Sommaire nella versione 2.1 è stato modificato per ridurre la maggioranza dei requisiti SQL. Una riscrittura completa del codice del blocco era necessaria completamente da ottimizzare il numero di richieste di SQL e da aggiungere le nuove caratteristiche.

Confronto
Il confronto è stato fatto in ogni situazione: se le pagine sono state viste da un visitatore (anonimo), un Membro, un Amministratore non loggato e loggato come membro.

  1. Blocco Originale Modules
    • Visitatori : 3 interrogazioni
    • Membri : 4 interrogazioni
    • Ammistratore : 81 interrogazioni
    • Administratore + Membro : 82 interrogazioni

  2. Sommaire Paramétrable v2.1
    • Visitatori : 4 interrogazioni
    • Membri : 6 interrogazioni
    • Amministratore : 7 interrogazioni
    • Amministratore + Membro : 9 interrogazioni
    Aggiunta 1 interrogazione per PHP-Nuke v7+ : Sommaire Paramétrable rende a gruppi di utenti l'autorizzazione per i moduli limitati.
    Aggiunte da 0 a 6 interrogazioni se il rilevamento delle "Novità" è attivato.

  3. Sommaire Paramétrable v2.0
    • Visitatori : 7 interrogazioni
    • Membri : 30 interrogazioni
    • Amministratore : appross. da 300 a 400 interrogazioni
    • Amministratore + Membro : appross.da 300 a 400 interrogazioni

Se non vuoi usare la rivelazione dei gruppi utenti (avviato con PHP-Nuke 7), puoi modificare la linea 43 in block-Sommaire.phpper rismarmiare 1 richiesta SQL (metti $gestiongroupe = 0).
Se non vuoi usare i 'Messaggi Privati', modifica il valore di $detectPM in block-Sommaire.php, linea 44 (metti 0) , per risparmiare 1 interrogazione SQL.


Come posso utilizzare dei "Links esterni" ?
I links estern possono essere facilmente creati inserendo un collegamento nel tuo menu.
Per usare la funzione, seleziona " *Links esterni* " (dove sono elencati I moduli), nel pannello amministativo di Sommaire.
Verranno generate due nuove caselle di testo, quando inserisci l'url del link e il nome del link, il collegamento esterno sarà presente nel menu.

Nei links esterni possono essere utilizzati tutti I tipi di url (link ad un file, streaming, etc...).

NOVITA' : Se il link punta ad una pagina specifica (parte di un modulo) del tuo sito, l'accesso a questo link nel menu può essere collegato a quel modulo.
Esempio : il modulo Downloads è riservato ai soli membri. il links esterno sarà : "modules.php?name=Downloads&d_op=viewdownload&cid=1" puntando alla categoria specifica del modulo "Downloads" che è invisibile ai visitatori anonimi.
Note : il link deve cominciare con "modules.php", non con "http://www.yoursite.com/modules.php" : è un link relativo.



Posso aprire un link in una nuova finestra di del browser ?
E' molto facile : per aprire un linkh in una nuova finestra, il collegamento deve cominciare per :
  • http://
  • https://
  • ftp://
  • link relativo (link ad una pagina specifica del tuo sito, esempio: "modules.php?name=Topics")
Per aprire un link nella stessa finestra, il collegamento deve cominciare per :
  • HTTP://
  • HTTPS://
  • FTP://

Per aiutarti a vedere rapidamente se il collegamento si aprirà o non in una nuova finestra, nel pannello amministrativo di Sommaire lo stato dell'icona cambia vicino al collegamento creato.


Posso aprire un link in una popup ?
Per aprire un link in una popup,  devi usare la funzione di "Links esterno".
Nella casella l 'url', deve contenere :

javascript:window.open('http://www.url.com','popup_sommaire','directories=no,menubar=no,status=no,location=no,scrollbars=no,resizable=no')

Ovviamente, devi sostituire il link www.url.com con quello da te desiderato...

Come uso altre immagini (icone) in Sommaire ?
Facile: aggiungi le tue immagini nella directory:
  • /images/sommaire/ per le immagini delle categorie
  • /images/sommaire/categories/ per le immagini di links/moduli
Le immagini appaiono automaticamente nella casella di selezione in sommaire admin.

Per modificare la 'NUOVA' icona,  puoi sostituire il file:
  • /images/sommaire/admin/new.gif
con l'immagine che preferisci.


Alcuni moduli compaiono nella parte inferiore del blocco Sommaire (non selezionato nell'admin Sommaire), voglio visulizzare anche quelli !
Con la versione 2.1, I moduli che non sono selezionati nell'admin sommaire sono visibili agli amministratori.
Sono visulizzati in 3 gruppi :
  • Moduli attivi e visibili, ma non selezionati: Questi moduli sono normalmente visulizzati nel menu, ma sono stati "dimenticati". Essi sono accessibili ai membri o agli anonimi se scrivono il link.
  • Moduli attivi e invisibili : Questi moduli non vengono visulizzati nel menu (puoi modificare i permessi nel pannello amministrativo : Cliccando su on 'Modifica' e rendi 'Visibile nel blocco-moduli').
  • Moduli inattivi : Questi moduli non sono accessibili nè a membri nè agli anonimi, quando vengono scritti gli url manualmente. Solo il Superadmins può vedere questi moduli.

Solo gli amministratori possono vedere I 3 gruppi di moduli, è totalmente invisibile a membri e anonimi.


Come posso selezionare l'individuazione automatica delle novità ?
Questa opzione, selezione nel blocco Sommaire I nuovi contenuti dei sguenti moduli:
  • Downloads
  • Web Links
  • Content
  • Reviews
  • Journal
  • News solo se il modulo è in homepage!
Le 'novità' sono individuate usando la 'data' nel campo delle tabelle corrispondenti al database .
Se un link esterno punta direttamente alla pagina di questi moduli, l'individuazione dei 'nuovi' contenuti è creato! (puoi facimente creare un link diretto ad alcune categorie di download, etc...)
(Il link deve cominciare con 'http://' or 'HTTP://', e con l'aggiunta di 'modules.php?name=')

Ho un messaggio di errore : Ho un messaggio di errore : 'Il PESO non è corretto. Il valore del PESO deve essere compreso tra 0 e 98!!', come posso risolvere il problema ?
Questo errore può avere differenti cause.
  • Il Peso indicato non è corretto !
    La prima causa, la più frequente è che hai utilizzato un Peso non valido per una categoria! Correggi e ricomincia and retry.

  • La tabella non è accessibile
    Il messaggio può apprire perchè la tabella Sommaire nel tuo database, non è accessibile.
    Il motivo potrebbe essere una cattiva installazione. Leggi il manuale di installazione (instruzioni nel file zip Sommaire : /manual-install/readme.txt)
    Se leggi il manuale modifica il prefix delle tue tabelle nel DB, nel file sql.

  • Hai installato un addon che crea conflitto interferendo con l'amministrazione di Sommaire
    Ci sono, solo 3 addons che hanno identificato questo problema :
    1. Shoutblock (Risolvi il problema)
    2. alcuni editor WYSIWYG (Risolvi il problema)
    3. la patch Php-Secure nel mainfile.php (a volte dipende anche dalla versione di PHP-Nuke installata) (Risolvi il problema)

    Leggi le relative FAQ per risolvere il problema.

Come creo un menu Multiligua ?
In Sommaire Paramétrable puoi creare un menu multilingua.
I visitatori che scelgono una differente lingua possono vedere il nome delle delle categorie e I links/moduli nella loro lingua !

Per fare ciò segui queste istruzioni:
  1. Nell'amministrazione di Sommaire, devi settare inomi delle categorie in questo modo :
    'LANG:_SOMCAT0' per la prima categoria,
    'LANG:_SOMCAT1' per la seconda categoria, etc...


  2. Alcuni moduli possono essere visulizzati con links esterni'.
    --> modifica l'indirizzo del modulo, per esempio 'modules.php?name=Name_of_Module'
    --> modifica il  testo del link con : 'LANG:_NameOfModule' (ex. 'LANG:_SOMSURVEYS')

  3. Qundo modifichi I files di lingua (website root /language/lang-xxx.php)
    --> aggiungi una nuova linea per la corretta visulizzazione, ad esempio :
    ex.
    define ("_SOMCAT0","Home");
    define ("_SOMCAT1","Content");
    define ("_SOMSURVEYS","Surveys");
    etc...
    --> Puoi copiare queste linee in tutti I files di lingua e tradurre il testo.
    ex. per francese può essere :
    define ("_SOMCAT0","Accueil");
    define ("_SOMCAT1","Contenu");
    define ("_SOMSURVEYS","Sondages");
    etc...

  4. Vai in Amministrazione->Moduli, e edita il modulo :
    rendendo "visibile nel blocco moduli".

  • Nessun problema con l'opzione di Menu Dinamico.

    • Problemi con Mozilla (versione testata : 1.6) : L'opzione di Menu Dinamico non è visualizzata correttamente, è automaticamente disabilitata per gli utenti Mozilla. L'amministrazione del pannello di controllo funziona correttamente.
